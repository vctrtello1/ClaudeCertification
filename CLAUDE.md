# UIGen — Claude Code Guide

## Project Overview

AI-powered React component generator with live preview. Users describe components in chat; Claude generates code via tool calls; changes appear instantly in an iframe preview.

## Commands

```bash
npm run setup        # Install deps + prisma generate + migrate (first-time setup)
npm run dev          # Dev server with Turbopack (localhost:3000)
npm run dev:daemon   # Dev server in background, logs → logs.txt
npm run build        # Production build
npm test             # Run vitest tests
npm run db:reset     # Reset database (destructive)
```

> The `NODE_OPTIONS='--require ./node-compat.cjs'` prefix is applied automatically by all npm scripts.

## Environment Variables

| Variable | Required | Notes |
|---|---|---|
| `ANTHROPIC_API_KEY` | No | Falls back to `MockLanguageModel` if absent |

## Architecture

### Key Directories

```
src/
  actions/          # Server Actions (auth, project CRUD)
  app/
    api/chat/       # POST /api/chat — AI generation endpoint
    [projectId]/    # Per-project page
    main-content.tsx
  components/
    auth/           # AuthDialog, SignIn/SignUpForm
    chat/           # ChatInterface, MessageList, MessageInput
    editor/         # FileTree, CodeEditor (Monaco)
    preview/        # PreviewFrame (sandboxed iframe)
    ui/             # shadcn/ui primitives
  lib/
    auth.ts         # JWT session management (jose)
    file-system.ts  # VirtualFileSystem class
    prisma.ts       # Prisma client singleton
    provider.ts     # getLanguageModel() — Anthropic or Mock
    anon-work-tracker.ts
    contexts/       # FileSystemContext, ChatContext
    prompts/        # System prompt for AI generation
    tools/          # str-replace.ts, file-manager.ts (AI tool definitions)
    transform/      # jsx-transformer.ts — Babel JSX → blob URLs
  hooks/
    use-auth.ts
  middleware.ts     # Protects /api/projects and /api/filesystem
```

### Virtual File System

`VirtualFileSystem` (`src/lib/file-system.ts`) is an in-memory tree — **no files are written to disk**. State is serialized to JSON and stored in `Project.data` (SQLite) on each chat completion.

- Entry point for preview: `/App.jsx` (or `/App.tsx`, `/index.jsx`, etc.)
- AI interacts with the VFS via two tools: `str_replace_editor` and `file_manager`
- `FileSystemContext` bridges AI tool calls → React state → editor/preview

### AI Generation Pipeline

1. Client (`ChatContext`) POSTs to `/api/chat` with `{ messages, files, projectId }`
2. Server deserializes `files` → `VirtualFileSystem`, calls `streamText()` with tools
3. Claude uses `str_replace_editor` (view/create/str_replace/insert) and `file_manager` (rename/delete)
4. Client receives streamed tool calls → `onToolCall` updates `FileSystemContext`
5. On finish: if authenticated, saves messages + VFS snapshot to DB

**Model:** `claude-haiku-4-5` via `@ai-sdk/anthropic`. Mock fallback generates Counter/Form/Card placeholders.

### JSX Transform / Preview

`createImportMap()` (`src/lib/transform/jsx-transformer.ts`):
1. Babel-transforms each `.jsx/.tsx` file (strips CSS imports, converts JSX)
2. Creates blob URLs for each transformed file
3. Builds an import map: `@/` alias + relative paths all point to blob URLs; third-party packages → `esm.sh`
4. Generates preview HTML injected into the sandboxed iframe

Tailwind CSS is loaded via CDN in the preview iframe. CSS files are injected as `<style>` tags.

### Auth

- JWT in `auth-token` cookie (httpOnly, 7-day lifespan)
- Server actions: `signUp`, `signIn`, `signOut`, `getUser` (`src/actions/index.ts`)
- Anonymous users are supported; work tracked in `sessionStorage` via `anon-work-tracker.ts`
- Middleware (`middleware.ts`) guards `/api/projects` and `/api/filesystem`

### Data Model

```prisma
User    { id, email, password, projects[] }
Project { id, name, userId?, messages (JSON), data (JSON), user? }
```

`userId` is nullable — anonymous projects have no owner.

## Testing

Tests use **Vitest** + **jsdom** + **@testing-library/react**.

```bash
npm test
```

Test files live next to source in `__tests__/` subdirectories. Path aliases (`@/`) are resolved via `vite-tsconfig-paths`.

## Code Conventions

- Path alias `@/*` maps to `src/*` (TypeScript + Vite)
- Components generated by AI must use Tailwind CSS (not inline styles) and import from `@/`
- `cn()` from `src/lib/utils.ts` for conditional class names (clsx + tailwind-merge)
- Server Actions are preferred over API routes for data mutations
- Prisma client is a singleton in `src/lib/prisma.ts`
